# Module: noema-fs

## Purpose

Handles all filesystem interactions: reading directories, performing file operations (copy/move/rename/delete) with undo support, and watching directories for external changes.

---

## Public Interface

### FileOpsEngine

```rust
pub struct FileOpsEngine {
    event_bus: Arc<EventBus>,
    db: Arc<Database>,
    undo_stack: Arc<Mutex<VecDeque<UndoableOp>>>,
    redo_stack: Arc<Mutex<VecDeque<UndoableOp>>>,
    max_undo_depth: usize,

impl FileOpsEngine {
    pub fn new(event_bus: Arc<EventBus>, db: Arc<Database>) -> Self;

    // Directory listing
    pub async fn list_directory(
        &self, 
        path: &Path, 
        sort: &SortOrder,
        show_hidden: bool,
    ) -> Result<Vec<FileEntry>, NoemaError>;

    // File operations (all return OperationId for progress tracking)
    pub async fn copy_files(
        &self,
        sources: Vec<PathBuf>,
        dest: PathBuf,
        on_conflict: ConflictStrategy,
    ) -> Result<OperationId, NoemaError>;

    pub async fn move_files(
        &self,
        sources: Vec<PathBuf>,
        dest: PathBuf,
        on_conflict: ConflictStrategy,
    ) -> Result<OperationId, NoemaError>;

    pub async fn delete_files(
        &self,
        paths: Vec<PathBuf>,
        use_trash: bool,
    ) -> Result<OperationId, NoemaError>;

    pub async fn rename_file(
        &self,
        path: PathBuf,
        new_name: String,
    ) -> Result<(), NoemaError>;

    pub async fn create_directory(
        &self,
        path: PathBuf,
    ) -> Result<(), NoemaError>;

    pub async fn create_file(
        &self,
        path: PathBuf,
    ) -> Result<(), NoemaError>;

    // Undo/Redo
    pub async fn undo(&self) -> Result<(), NoemaError>;
    pub async fn redo(&self) -> Result<(), NoemaError>;
    pub fn can_undo(&self) -> bool;
    pub fn can_redo(&self) -> bool;

    // Metadata
    pub async fn get_metadata(&self, path: &Path) -> Result<FileMetadata, NoemaError>;
}

pub enum ConflictStrategy {
    Ask,        // emit event asking frontend for decision
    Skip,
    Overwrite,
    Rename,     // append (1), (2), etc.
}

pub struct FileMetadata {
    pub entry: FileEntry,
    pub permissions: Permissions,
    pub owner: Option<String>,
    pub group: Option<String>,
    pub inode: Option<u64>,
    pub blocks: Option<u64>,
    pub content_hash: Option<String>,   // BLAKE3, computed on demand
}
```

## Behavior

### Directory Listing
- Returns entries sorted by `SortOrder`
- Directories always listed first (configurable)
- Hidden files filtered based on `show_hidden` flag
- Symlinks resolved for metadata but displayed as symlinks
- Broken symlinks included with error indicator
- Permission-denied entries included with error flag (not skipped silently)

### Copy
1. Validate all source paths exist
2. Validate destination is a directory
3. Emit `OperationStarted`
4. For each source:
   - If file: copy bytes with progress
   - If directory: recursive copy preserving structure
   - On conflict: apply `ConflictStrategy`
5. Push `UndoableOp::Copy` with list of created files
6. Emit `OperationComplete`
7. Emit `FileChanged::Created` for each new file

### Move
1. Same-volume: `std::fs::rename` (atomic)
2. Cross-volume: copy + verify + delete original
3. Push `UndoableOp::Move` with (src, dest) pairs
4. Emit `FileChanged::Renamed` for each moved file

### Delete
- If `use_trash`: move to platform trash (via `trash` crate)
- If permanent: remove with confirmation already obtained from frontend
- Push `UndoableOp::Delete` with trash recovery info
- Emit `FileChanged::Deleted` for each

### Rename
- Validate new name (no path separators, no null bytes, not empty)
- Validate no conflict at destination
- `std::fs::rename` (same directory, atomic)
- Push undo entry
- Emit `FileChanged::Renamed`

### Undo
- Pop from undo stack
- Reverse the operation:
  - Copy → delete created files
  - Move → move back
  - Delete → restore from trash (if trashed) or error (if permanent)
  - Rename → rename back
- Push reversed op to redo stack
- Emit appropriate `FileChanged` events

### Watcher
- Uses `notify` crate with `RecommendedWatcher`
- Debounce window: 500ms (batch rapid changes)
- Ignores paths matching `.noemaignore` patterns
- On event: emit `FileChanged` to event bus
- Correlates delete+create within debounce window as rename (by checking file hash if available)

## Edge Cases & Error Handling

- **Source disappeared mid-copy:** Emit error event, skip that file, continue others, report partial success
- **Destination full:** Emit error, clean up partial copy, report failure
- **Permission denied on single file in batch:** Skip with warning, continue others
- **Rename conflict:** Return error to frontend, let user decide
- **Undo of permanent delete:** Return `NoemaError::Cancelled` with explanation (cannot undo permanent delete)
- **Circular symlinks:** Detect via visited-set during recursive copy, skip with warning
- **File locked by another process (Windows):** Retry once after 100ms, then report error
- **Watcher overflow (too many events):** Trigger full re-scan of affected directory
- **Network drive goes offline:** Detect I/O error, mark watched path as offline, retry periodically

## Performance Constraints

- `list_directory`: <50ms for 10k entries, <500ms for 100k entries
- Copy throughput: ≥200MB/s (limited by disk, not our code)
- Watcher event delivery: <500ms from filesystem change to `AppEvent` emission (debounce included)
- Undo/redo: <100ms (metadata-only operation)

## Example Usage

```rust
let fs_engine = FileOpsEngine::new(event_bus.clone(), db.clone());

// List a directory
let entries = fs_engine.list_directory(
    Path::new("/Users/me/Documents"),
    &SortOrder { field: SortField::Modified, direction: SortDirection::Desc },
    false,
).await?;

// Copy files
let op_id = fs_engine.copy_files(
    vec!["/Users/me/report.pdf".into()],
    "/Users/me/backup/".into(),
    ConflictStrategy::Rename,
).await?;

// Undo last operation
fs_engine.undo().await?;

// Start watcher
let mut watcher = WatcherService::new(event_bus.clone(), config.indexing)?;
watcher.start().await?;
```
