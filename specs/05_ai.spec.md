# Module: noema-ai

## Purpose

Manages local LLM inference for context generation (summaries, entity extraction, tag suggestions, smart rename) and provides the embedding engine interface used by the indexer and search modules.

---

## Public Interface

### LLM Engine

```rust
pub struct LlmEngine {
    model: Option<LlamaModel>,
    config: AiConfig,
}

impl LlmEngine {
    pub fn new(config: &AiConfig) -> Self;
    
    pub async fn load_model(&mut self) -> Result<(), NoemaError>;
    pub fn unload_model(&mut self);
    pub fn is_loaded(&self) -> bool;

    pub async fn generate_context(
        &self,
        content: &str,
        max_tokens: usize,
    ) -> Result<GeneratedContext, NoemaError>;

    pub async fn suggest_filename(
        &self,
        content: &str,
    ) -> Result<String, NoemaError>;

    pub async fn suggest_tags(
        &self,
        content: &str,
        existing_tags: &[String],
    ) -> Result<Vec<String>, NoemaError>;

    pub async fn extract_entities(
        &self,
        content: &str,
    ) -> Result<Vec<Entity>, NoemaError>;

    pub async fn summarize_multi(
        &self,
        contents: &[(&str, &str)],   // (filename, content) pairs
    ) -> Result<String, NoemaError>;
}

pub struct GeneratedContext {
    pub summary: String,
    pub entities: Vec<Entity>,
    pub suggested_tags: Vec<String>,
    pub key_phrases: Vec<String>,
}

pub struct Entity {
    pub text: String,
    pub entity_type: EntityType,
}

pub enum EntityType {
    Person,
    Organization,
    Location,
    Date,
    Concept,
    Technology,
    Other(String),
}
```

    pub generated_at: DateTime<Utc>,

## Behavior

### Model Lifecycle

- Model is **not** loaded on app start (saves 500MB+ RAM)
- Loaded lazily on first `generate_context` call
- Unloaded after 2 minutes of inactivity (configurable)
- Only one inference runs at a time (dedicated thread, sequential queue)
- If model loading fails: return `NoemaError::Ai` with clear message, app continues without AI

### Context Generation Prompt

```
System: You are a file analysis assistant. Given a document's content, produce:
1. A 2-3 sentence summary of what this document is about
2. Key entities (people, organizations, dates, concepts)
3. 3-5 suggested tags for categorization
4. Key phrases that capture the main topics

Respond in JSON format:
{
  "summary": "...",
  "entities": [{"text": "...", "type": "person|org|location|date|concept|tech"}],
  "tags": ["...", "..."],
  "key_phrases": ["...", "..."]
}

Document content:
{first 4000 tokens of document}
```

### Smart Rename

```
Given this document content, suggest a descriptive filename (no extension).
Use format: YYYY-MM-DD_descriptive_name (if date is relevant) or just descriptive_name.
Use lowercase, underscores, no special characters.
Content: {first 2000 tokens}
```

### Context Versioning

- Each generation creates a new version (never overwrites)
- User edits create a new version with `user_edited = true`
- Max 10 versions kept per file (oldest auto-pruned)
- Latest version is what's displayed and searched

### Auto-Generate (Optional)

- When `auto_generate_context = true` in config:
  - After indexing completes for a file, queue context generation
  - Low priority (runs after all indexing is done)
  - Skip files that already have context (unless content changed)

## Edge Cases & Error Handling

- **Model file not found:** Return friendly error suggesting model download/setup
- **Content too long for context window:** Truncate to first N tokens, note truncation in summary
- **LLM produces invalid JSON:** Attempt to extract partial fields via regex fallback
- **LLM generates hallucinated entities:** User can remove via edit; feedback loop (future)
- **Inference interrupted (app quit):** No partial save; generation retried on next request
- **Multiple generate requests for same file:** Deduplicate in queue, return same future

## Performance Constraints

- Model load: <5s (cold), <500ms (warm, already in disk cache)
- Context generation (50-page doc): <10s on mid-tier CPU (M1/Ryzen 5)
- Context generation (1-page doc): <2s
- Smart rename suggestion: <3s
- Entity extraction: included in context generation (no extra call)
- Memory: model loaded ~500MB–2GB depending on quantization

## Example Usage

```rust
let ai = LlmEngine::new(&config.ai);

// Generate context for a file
let content = std::fs::read_to_string("report.pdf.txt")?; // pre-parsed text
let context = ai.generate_context(&content, 512).await?;

println!("Summary: {}", context.summary);
println!("Tags: {:?}", context.suggested_tags);

// Save to store
let store = ContextStore::new(db.clone());
store.save_context(file_id, &context).await?;

// Smart rename
let suggestion = ai.suggest_filename(&content).await?;
println!("Suggested name: {}.pdf", suggestion);
```
